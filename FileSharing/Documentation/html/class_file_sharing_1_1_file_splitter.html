Splits the file lazily into multiple subfiles.

This requires that you have permission to write on the system. If you only need the bytes and your file is smaller than whats allowed by the clr use SplitFile(string, int) instead

Parameters
filePathThe path to the file.
chunkSizethe size of the chunks to split the files into
splitFilesDirectorythe folder where to store the files. if the folder does not exist, then it will created. Default is the current directory
splitFilesNamethe name of the splitfiles. For example if splitFilesName = "Part" then the files will be named "Part1.split","Part2.split" and so on
isPathAbsolouteWhether the returned paths to the files are absoloute or relative to the splitFilesDirectory
Returns
An enumerable of all the paths to the split files.

This example shows how to use this method

//Splits the test.jpg into 50kb chunks and outputs them into the current directory
var files = SplitFile("Test.jpg", 1024 * 50);
foreach (var item in files)
Console.WriteLine(item);
/* Prints
* Path
* Part1.split
* Part2.split
* Part3.split
* And so on
*/
